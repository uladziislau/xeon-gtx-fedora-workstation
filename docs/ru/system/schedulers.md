# 🧠 Sched-ext: Выбор Планировщика для 36 Потоков

> Здесь задокументирован опыт использования различных eBPF-планировщиков (`sched_ext`) на ядре CachyOS. Задача: найти баланс между отзывчивостью (latency) и пропускной способностью (throughput) на архитектуре NUMA.

## Хронология Тестов

### 1. SCX_LAVD (Latency-criticality Aware Virtual Deadline)
*   Статус: ❌ **КРАХ**
*   Симптомы: Kernel panic / NULL pointer dereference.
*   Причина: В режиме "Autopilot" LAVD перегрузил нулевое ядро (Core 0), пытаясь управлять всеми прерываниями через него. Система захлебнулась.
*   Вердикт: Не подходит для текущей конфигурации 18-ядерного Xeon без тонкой ручной настройки масок.

### 2. SCX_RUSTY
*   Статус: ⚠️ **Условно годен**
*   Плюсы: Отличная многозадачность, использует все ядра.
*   Минусы: При запуске тяжелых приложений (браузер с 30+ процессами) курсор мыши начинает "лагать". Rusty ориентирован на серверные нагрузки (throughput), а не на десктопную плавность.

### 3. SCX_RUSTLAND
*   Статус: ✅ **Выбор для Плавности**
*   Плюсы: Абсолютный приоритет интерактивных задач (KWin, Xwayland, Браузер). Курсор мыши не лагает никогда, даже при 100% загрузке CPU.
*   Минусы: Чуть меньшая скорость в синтетических бенчмарках.
*   Инцидент: Было подозрение на зависание системы при просмотре видео, но вероятная причина — конфликт драйвера NVIDIA, а не планировщика.

### 4. SCX_BPFLAND
*   Статус: ⭐ **Текущий Фаворит**
*   Описание: Простой и надежный планировщик vtime.
*   Результат: "Весьма нравится". Субъективно дает самую стабильную работу системы без крайностей.

## Рекомендация
Для повседневной работы на десктопе использовать **scx_bpfland** или **scx_rustland**. Если важен каждый FPS в рендеринге (Blender) — переключаться на **scx_rusty**.
